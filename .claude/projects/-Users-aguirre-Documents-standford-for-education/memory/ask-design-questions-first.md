When Eduardo requests a significant redesign of the Lemonville simulation (or similar), he expects clarifying design questions BEFORE code is written — he explicitly invited them ("ask me if you have any questions about the design") when rejecting the first draft.

**Why:** The v3→v4 pivot (duopoly → 4-tier ecosystem) hinged on four choices he wanted to make himself: role layout (multiple students per tier), time model (synchronized rounds), butterfly visualization (animated cascade on the town map), and scoring (goal + decision quality + ecosystem impact as three separate scores).

**How to apply:** Ground questions in the existing code first, present 2–4 concrete options with trade-offs (previews help), and record the locked decisions in `lemonville-build-spec.md` so later sessions don't re-litigate them.
